[쿠키]
    1. 정의
        클라이언트(브라우저)의 임시 저장소로,
        사용자의 로그인 정보나 설정값 등을 작은 데이터 형태로 저장하는 기술

    2. 목적
        로그인 상태 유지 : 사용자가 새로고침하거나 다른 페이지로 이동해도 로그인 유지
        사용자 맞춤 서비스 : 테마, 언어, 장바구니 등 개인 설정 저장
        트래킹 및 분석 : 방문자 행동, 통계 분석에 활용

    3. 비교 (세션 vs 쿠키)
        구분	            저장 위치             관리주체	        유효 기간	            보안성	    예시
        세션(Session)	서버	                서버	            서버 종료 시 삭제	    높음 🔒	    관리자 페이지 로그인
        쿠키(Cookie)	    클라이언트(브라우저)	클라이언트	    설정된 기간 유지	    낮음 ⚠️	    자동 로그인, 장바구니

        💡 정리 :
        세션은 서버에 저장 → 보안은 높지만 서버 부하 증가
        쿠키는 클라이언트에 저장 → 서버 부담은 적지만 보안에 주의 필요

    4. 쿠키 용어 및 속성
        속성	설명	예시
        name	                쿠키 이름	"loginUser"
        value	                쿠키 값 (저장할 데이터)	"tj1234"
        setHttpOnly(true)	    JS로 접근 불가능 (보안 강화)	민감정보 필수 설정
        setSecure(true)	        HTTPS 환경에서만 쿠키 사용	실제 배포 시 true
        setPath("/")	        접근 가능한 경로 지정	/ → 전체 경로에서 접근 가능
        setMaxAge(초)	        쿠키의 유효시간	60 * 60 → 1시간
        addCookie()	            응답에 쿠키를 담아 클라이언트로 전송	response.addCookie(cookie)

6. 쿠키 보안 주의사항 ⚠️
    민감정보 저장 금지 : 비밀번호, 토큰 등은 절대 저장하지 않음
    HttpOnly 설정 필수 : 자바스크립트로 접근 차단
    Secure 설정 : HTTPS 환경에서만 쿠키 송수신
    만료시간 설정 : 오래된 쿠키로 인한 세션 혼동 방지

7. 한눈에 보는 쿠키 흐름도
    [1] 로그인 성공 → 쿠키 생성 → 클라이언트에 저장
    [2] 페이지 이동 시 → 쿠키 자동 전송
    [3] 서버는 쿠키를 읽어 로그인 상태 유지
    [4] 로그아웃 → 쿠키 삭제
