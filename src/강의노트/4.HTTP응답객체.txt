[ HTTP 응답 코드 ]
1. 정의
    서버(응답자)가 클라이언트(요청자)의 요청 처리 결과를 숫자로 나타내는 표준 코드

2. 구간별 주요 코드
    ✅ 1xx (정보)
        100 Continue : 요청의 일부를 받았으며 나머지를 계속 전송하라는 의미
        101 Switching Protocols : 프로토콜 전환 승인 (ex. HTTP → WebSocket)

    ✅ 2xx (성공)
        200 OK : 요청 성공, 응답 데이터 포함
        201 Created : 생성/저장 성공, 응답 데이터 포함
        202 Accepted : 요청 수락했으나 아직 처리 완료는 아님 (비동기 작업 등)
        204 No Content : 요청 성공, 응답 데이터 없음

    ✅ 3xx (리다이렉션)
        301 Moved Permanently : 요청 리소스가 영구적으로 다른 URL로 이동
        302 Found : 임시적으로 다른 URL 사용
        304 Not Modified : 캐시 사용 가능, 변경 없음

    ✅ 4xx (클라이언트 오류)
        400 Bad Request : 요청 형식 오류 (쿼리, Body, 매핑 등 잘못됨)
        401 Unauthorized : 인증 필요 (로그인 안 됨)
        402 Payment Required : 결제 필요 (예약된 코드, 일부 API에서 사용)
        403 Forbidden : 접근 권한 없음 (권한 거부)
        404 Not Found : 요청한 리소스 없음 (경로/데이터 없음)
        405 Method Not Allowed : 요청한 HTTP 메서드 사용 불가 (예: GET만 허용인데 POST 요청)
        409 Conflict : 요청 충돌 (예: 아이디 중복, 자원 충돌)
        410 Gone : 리소스가 서버에서 영구적으로 삭제됨
        415 Unsupported Media Type : 지원하지 않는 Content-Type 요청
        429 Too Many Requests : 클라이언트 요청이 너무 많음 (Rate Limit 초과)

    ✅ 5xx (서버 오류)
        500 Internal Server Error : 서버 내부 오류
        501 Not Implemented : 서버가 해당 요청 처리 기능을 지원하지 않음
        502 Bad Gateway : 게이트웨이/프록시 서버가 잘못된 응답을 받음
        503 Service Unavailable : 서버 과부하, 점검 중 등으로 서비스 불가
        504 Gateway Timeout : 게이트웨이/프록시에서 응답 시간 초과

200/201/204, 400/401/403/404/405/409, 500/502/503

[ 스프링 에서 응답 코드 전송/반환 ]
    📌 Spring에서 응답 코드 반환
        1. ResponseEntity<T>
        Spring에서 HTTP 응답 상태 코드 + 본문을 함께 제어하는 클래스
        <T>는 반드시 참조 타입 사용 (기본 타입 불가)

    2. 사용 형태
    ResponseEntity<반환타입> 메소드명() {
        return ResponseEntity.status(응답코드).body(응답값);
    }

    3. 방법
    본문 포함 응답
    return ResponseEntity.status(200).body("OK");
    본문 없는 응답
    return ResponseEntity.status(204).build();





    1. 정의 : 응답자(서버)가 요청자(클라이언트)의 요청에 대한 결과를 나타내는 코드
    2. 종류 :
        1xx : 요청을 받고 프로세스 계속 진행중.
        2xx : 요청을 받고 응답을 이해, 처리 완료
            [주로 많이 발생하는 코드]
            200 : OK , 요청 성공 하고 응답 포함한다.
            201 : Create(저장)요청 성공 하고 응답 값을 포함한다.
            204 : 요청 성공 하고 응답 값 포함하지 하지 않는다.
        3xx : 요청을 완료하기 추가 조치 요구
        4xx : ( 클라이언트 오류 )
            [주로 많이 발생하는 코드]
            400 : 서버가 이해할수 있는 요청    ( 쿼리스트링,body,매핑 등 오류 )
            401 : 서버가 요청에 따른 권한 없음
            403 : 서버가 요청에 따른 권한 거부
            404 : 서버가 요청에 따른 리소스(값/html/매핑 등) 찾을수 없을떄
            409 : 서버가 요청에 따른 값이 중복 일때 ( 아이디중복 등 )
        5xx : ( 서버 오류 )
            500 : 서버의 예기치 못한 오류 발생

[ 스프링 에서 응답 코드 전송/반환 ]
    1. ResponseEntity<T> : Spring 에서 HTTP 응답 객체를 제어할 수 있게 해주는 클래스
        * 참고 : < > 제네릭 타입은 기본타입 이 불가능하고 참조타입 가능하다.
    2. 형태
        ResponseEntity< 반환타입 > 메소드명( ) {
            return ResponseEntity.status( 응답코드 );
        }
    3. 방법
        1. ResponseEntity.status( 응답코드 ).body( 반환값 );
        2. ResponseEntity.status( 응답코드 ).build()
